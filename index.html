Layout

  • Cuando se establece en relativa, la posición de un elemento es relativa a su posición predeterminada en la página.
  • Cuando se establece en absoluto, la posición de un elemento es relativa a su elemento principal posicionado más cercano. Se puede anclar a cualquier parte de la página web, pero el elemento aún se moverá con el resto del documento cuando se desplace la página.
  • Cuando se establece en fijo, la posición de un elemento se puede fijar a cualquier parte de la página web. El elemento permanecerá a la vista pase lo que pase.
  • Cuando se establece en fijo, un elemento puede quedarse en una posición de desplazamiento definida cuando el usuario desplaza su contenedor principal.
  • El índice z de un elemento especifica qué tan atrás o qué tan adelante aparece un elemento en la página cuando se superpone a otros elementos.
  • La propiedad de visualización le permite controlar cómo fluye un elemento vertical y horizontalmente en un documento.
  • Los elementos en línea ocupan el menor espacio posible y no pueden ajustarse manualmente el ancho o el alto.
  • Los elementos de bloque ocupan el ancho de su contenedor y pueden tener alturas ajustadas manualmente.
  • Los elementos de bloque en línea pueden tener un ancho y un alto establecidos, pero también pueden aparecer uno al lado del otro y no ocupan todo el ancho del contenedor.
  • La propiedad flotante puede mover elementos tanto a la izquierda como a la derecha en una página web.
  • Puede borrar el lado izquierdo o derecho de un elemento (o ambos) utilizando la propiedad clear.

Teoria del color

Las claves para elegir los colores adecuados para sus proyectos son:

>
  • Usar la rueda de colores como base para seleccionar colores
  • Usar un enfoque de combinación de colores que promueva la armonía
  • Usar colores que se ajusten al contexto y la emoción que intenta mostrar a los usuarios
  • Usar el contraste para mejorar la legibilidad de los elementos de la página
  • Usar sombras y tintes de un color para crear flexibilidad
  • Evitar combinaciones que puedan causar problemas a los usuarios.

Recuerde, las claves para diseñar para UI:

>
  • Seleccione un color de marca dominante y colores de acento de apoyo
  • Usa el contraste para definir secciones y diferenciar acciones.
  • Utilice colores semánticos para los mensajes de error y éxito
  • Incorporar colores predeterminados para texto y fondos donde sea necesario
  • Los colores neutros pueden proporcionar un buen contraste.
  • Tenga en cuenta a los usuarios al seleccionar el color

Diseño del Texto

  • Diferenciar el texto es esencial y hay varias formas de hacerlo:
    • Utilice elementos de encabezado h1, h2 y h3
    • Utilice 2-3 fuentes bien emparejadas, pero no más.
    • Ajuste el estilo, el tamaño, el peso, el espaciado y el color de la fuente.
  • El espacio en blanco juega un papel importante en la legibilidad. Asegúrese de que todos sus elementos de texto tengan suficiente espacio en blanco.
  • Utilice la convención estándar para los vínculos y los botones de navegación para que los usuarios sepan qué esperar.
  • Utilice columnas cuando sea necesario para mantener la longitud de la línea entre 50 y 75 caracteres.
  • Utilice tarjetas de colores para emparejar imágenes con texto importante de modo que los usuarios se sientan atraídos hacia él.
  • Tenga los elementos que desea que los usuarios noten en la parte superior de la página o en el lado izquierdo porque esas son las áreas de la página que los usuarios notarán al hojear.

Botones y links

  • Se agregaron estilos para que los enlaces sean reconocibles y distinguibles del texto ordinario.
  • Se agregaron estilos de enlace que dependen del estado de interacción del mouse, lo que brinda a los usuarios comentarios visuales para el desplazamiento del cursor y los clics del mouse.
  • Se agregó texto de contexto adicional con el atributo de título HTML.
  • Botones con estilo para ser fácilmente reconocibles (en estilos de diseño esquemórfico y plano) usando formas de cuadro, borde, desplazamiento y estados activos.

Navegación secundaria

  • Utilice rutas de navegación para indicar dónde se encuentra un usuario y la extensión del sitio.
  • Las rutas de navegación se implementan mediante listas desordenadas en HTML con estilo CSS personalizado
  • Existen tres tipos de migas de pan:
    • ubicación - basada en la estructura jerárquica del sitio
    • atributo: basado en los atributos de la página o artículo actual
    • ruta: única para el recorrido de un usuario en el sitio
  • Las migas de pan basadas en rutas pueden ser confusas, solo úselas si es necesario
  • Asegúrese de que las migas de pan agreguen valor antes de agregarlas a un sitio

Responsive

  • El contenido de un sitio web se puede dimensionar en relación con otros elementos de la página utilizando medidas relativas.
  • La unidad de fuente de tamaños de em relativa al tamaño de fuente de un elemento principal.
  • La unidad de fuente de tamaños rem en relación con el tamaño de fuente de un elemento raíz. Ese elemento raíz es el elemento .
  • Los porcentajes se utilizan comúnmente para dimensionar las características del modelo de caja, como el ancho, la altura, el relleno o el margen de un elemento.
  • Cuando se utilizan porcentajes para dimensionar el ancho y la altura, los elementos secundarios se dimensionarán en relación con las dimensiones de sus principales (recuerde que las dimensiones principales deben establecerse primero).
  • Los porcentajes se pueden utilizar para establecer el relleno y el margen. El margen y el relleno horizontal y vertical se establecen en relación con el ancho de un elemento principal.
  • El ancho mínimo y máximo de los elementos se puede establecer utilizando el ancho mínimo y el ancho máximo.
  • La altura mínima y máxima de los elementos se puede establecer utilizando la altura mínima y la altura máxima.
  • Cuando se establece la altura de una imagen o video, entonces su ancho se puede establecer en automático para que los medios escalen proporcionalmente. La inversión de estas dos propiedades y valores también logrará el mismo resultado.
  • Una imagen de fondo de un elemento HTML se escalará proporcionalmente cuando su propiedad de tamaño de fondo esté configurada para cubrir.

Media Query

  • Cuando un sitio web responde al tamaño de la pantalla en la que se ve, se denomina sitio web adaptable.
  • Puede escribir consultas de medios para ayudar con diferentes tamaños de pantalla.
  • Las consultas de medios requieren funciones de medios. Las características de los medios son las condiciones que deben cumplirse para representar el CSS dentro de una consulta de medios.
  • Las funciones multimedia pueden detectar muchos aspectos del navegador de un usuario, incluido el ancho, la altura, la resolución, la orientación y más de la pantalla.
  • El operador y requiere que varias funciones multimedia sean verdaderas a la vez.
  • Una lista separada por comas de características multimedia solo requiere que una característica multimedia sea verdadera para que se aplique el código que contiene.
  • La mejor práctica para identificar dónde se deben establecer las consultas de medios es cambiar el tamaño del navegador para determinar dónde se rompe el contenido de forma natural. Los puntos de interrupción naturales se encuentran cambiando el tamaño del navegador.

Flex

  • display: flex -cambia el elemento a modo bloque y puede contener objetos flex en su int.
  • display: inline-flex -permite a los contenedores aparecer en linea.
  • justify-content -se usa para colocar elementos en el axis horizontal.
  • align-items -es usado para espaciar los objetos en el axis vertical.
  • flex-grow -Se usa para especificar cuando espacio (y proporciones) lo que un objeto flex puede absorber en horizontal.
  • flex-shrink -Se utiliza para especificar cuánto se encogen los elementos flexibles y en qué proporciones a lo largo del eje horizontal.
  • flex-basis -se utiliza para especificar el tamaño inicial de un elemento diseñado con flex-grow y/o flex-shrink.
  • flex -se utiliza para especificar flex-grow, flex-shrink y flex-basis en una declaración.
  • flex-wrap -especifica que los elementos deben desplazarse a lo largo del eje vertical si el contenedor flexible no es lo suficientemente grande(ancho).
  • align-content -se utiliza para espaciar filas a lo largo del eje vertical.
  • flex-direction -se utiliza para especificar los ejes horizontal y vertical.
  • flex-flow -se utiliza para especificar flex-wrap y flex-direction en una declaración.
  • Los contenedores flexibles se pueden anidar uno dentro de otro declarando display: flex o display: inline-flex para hijos de contenedores flexibles.
